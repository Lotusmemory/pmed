# 项目：PubMed 文献检索与分析 Demo（全栈开发）

你要为我创建一个完整的网页应用 Demo，用于 PubMed 文献检索、统计、可视化与 AI 综述生成。请严格按以下需求和规则进行开发，优先保证功能可用性和 UI 美观度。

## 一、项目背景与目标
- 公司：沿长科技（成都）有限公司
- 岗位：全栈开发工程师笔试
- 用户输入一个生物医学关键词 → 调用 PubMed API 检索文献 → 在前端展示：
  1. 数据统计卡片（文献总数、年份分布、期刊分区、影响因子概况）
  2. 可视化图表（词云、研究方向总结图）
  3. 近 5 年影响因子前 100 篇文献列表
  4. 基于摘要生成的约 500 字中文分点综述报告
- 要求现场可演示、可测试，具备较好的 UI 设计和交互体验。

## 二、技术栈要求（你可按此技术栈生成代码，也可提出更好方案）
- 前端：React + TypeScript + Tailwind CSS，使用 Vite 构建
- 图表库：ECharts（用于研究方向图表、年份分布等）
- 词云：使用 `react-wordcloud` 或 `wordcloud2.js`，或后端 Python 生成词云图片返回
- 后端：Python 3 + FastAPI，提供 RESTful API
- PubMed 数据获取：使用 `biopython` 的 `Entrez` 模块，或直接构造 eUtils API 请求
- LLM 调用：通过 HTTP 调用兼容 OpenAI 接口的大模型（可在配置中修改 api_base 和 api_key）
- 影响因子数据：因 PubMed 不直接提供影响因子，请在后端内建一个期刊 IF 映射表（JSON 文件，包含至少 200 个常见生物医学期刊及其 2024 影响因子，可参考公开数据自行构造或使用模拟合理值）。对于未匹配的期刊，随机分配 0.5–10 之间的 IF 值。

## 四、详细功能需求

### 1. 首页搜索界面
- 居中搜索框，带有醒目的搜索按钮，支持按 Enter 搜索。
- 搜索框下方可放置高级选项（可折叠）：出版日期范围、文献类型（默认所有）、排序方式。
- 加载状态：搜索时显示骨架屏或加载动画，各模块渐进式呈现。

### 2. 数据统计模块
- 检索完成后，在页面顶部展示统计卡片：
  - 总文献数量
  - 覆盖的年份范围
  - 平均影响因子
  - 涉及的一区/二区期刊数量（分区规则：IF >= 10 为一区，5-10 为二区，2-5 为三区，<2 为四区；可自定义）
- 这些数据需基于实际检索结果中的期刊 IF 计算。

### 3. 可视化模块
- **词云**：根据检索结果的标题+摘要提取高频关键词（去除停用词），生成词云图。建议在后端使用 Python `wordcloud` 库生成图片并返回 base64 给前端，或前端使用库直接生成。
- **研究方向总结图表**：可使用 ECharts 展示：
  - 每年发文数量柱状图
  - 期刊分布饼图（Top 10 期刊）
  - 研究热点主题的树图或矩形树图（基于 MeSH 词或关键词聚类）。若无法实时聚类，可使用 LDA 主题模型或简单 TF-IDF 提取前 10 个主题词展示。

### 4. 高影响力文献列表
- 从检索结果中筛选出 **近 5 年**（2021-2026）且 **影响因子排名前 100** 的文献。
- 以表格形式展示：序号、标题、作者、期刊、年份、影响因子、PMID、DOI 链接。
- 支持点击标题展开摘要预览。
- 表格需支持排序和分页（每页 20 条）。

### 5. AI 综述报告生成
- 收集前 100 篇高影响力文献的摘要（如果少于 100 篇，则用全部）。
- 调用大模型 API，系统提示词如下：
你是一位资深的生物医学领域研究员，请根据以下文献摘要，撰写一份约 500 字的中文分点综述。要求：

概括该研究领域的最新进展和热点方向

分点阐述，每个点有小标题，语言严谨但通俗

指出存在的挑战或未来趋势

仅输出综述内容，不要包含其他解释

text
- 将摘要拼接进 user 消息，发给 LLM。
- 前端展示生成的综述，支持复制和重新生成。
- LLM API 配置放置于后端 `.env` 文件，需包含：
- `LLM_API_BASE`（默认为 `https://api.openai.com/v1`）
- `LLM_API_KEY`（由用户填写）
- `LLM_MODEL`（默认 `gpt-4o-mini`）
- 调用时需处理超时和错误，若失败在前端给出友好提示，并可选择使用本地规则模板生成一段简单的示例综述。

### 6. 错误处理与边界情况
- PubMed 无搜索结果时，显示“未找到相关文献，请更换关键词”。
- API 调用失败时，显示错误信息并建议重试。
- LLM 不可用时，综述区域展示“综述生成模块未配置或暂时不可用”。

## 五、UI/UX 设计要求
- 整体风格：现代、简洁、学术感，色调以蓝白为主，可使用 Tailwind 的蓝色系配色。
- 响应式布局：在桌面端和移动端均可良好显示。
- 各模块之间用卡片分区，带有平滑的微交互和过渡动画。
- 搜索历史：自动保存最近 5 次搜索关键词在本地，点击可快速填充。
